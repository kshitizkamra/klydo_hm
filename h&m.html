Technical Integration
HM NOW: End-to-End Process Flow

Intelligent Cataloging

Step 1

Direct pull of catalog data from H&M to Klydo/Fynd daily — or manual upload of new styles via Excel.

Purchase Order / Stock Transfer

Step 2

Klydo shares a PO/STN to move stock to the Klydo Darkstore. Inwards processed on Fynd — products go live on HMNow.

Omni variant: H&M store set up as darkstore in Fynd. POS integrated for styles & stock. Store sales pushed via API, reducing stock from Fynd in real-time.

❯

Location-Based Discovery via APIs

Step 3

API architecture ensures HMNow users only see products available at the Bangalore darkstore. "Immediate delivery" badge appears when stock is present.

Multi-source Discovery: Feed, collections, search, category navigation

Real-Time Inventory: In-stock items surfaced dynamically

Personalized Browsing: Tailored to regional trends & local stock

Integrated Order Management

Step 4

Orders transmitted to Fynd's OMS for fulfillment. Klydo manages warehouse pick & pack. Webhooks push shipment and delivery events back to HMNow.

Omni variant: Order instantly pushed to H&M POS — stock reduced automatically. Store staff packs and labels the order separately.

❯

Last-Mile Delivery & Real-Time Updates

Step 5

Last-mile logistics handled by Fynd. Klydo consumes tracking data to give full order visibility to the customer.

Order Picked Up: Status update triggered to HMNow via API

Live Tracking: Customer monitors rider on interactive map

Secure Delivery: OTP forwarded white-labeled as HMNow

Post-Order Returns Flow

Step 6
1

User requests return

2

Return + reverse shipment created in Fynd

3

OTP sent to customer (white-labeled as HMNow)

4

Delivery executive picks up → brought to warehouse

5

Quality check passed → Klydo webhooks H&M → Refund initiated by HMNow
