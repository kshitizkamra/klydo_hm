Technical Integration
HM NOW: End-to-End Process Flow

Intelligent Cataloging

Step 1

Direct pull of catalog data from H&M to Klydo/Fynd daily — or manual upload of new styles via Excel.

Purchase Order / Stock Transfer

Step 2

Klydo shares a PO/STN to move stock to the Klydo Darkstore. Inwards processed on Fynd — products go live on HMNow.

❯

Location-Based Discovery via APIs

Step 3

API architecture ensures HMNow users only see products available at the Bangalore darkstore. "Immediate delivery" badge appears when stock is present.

Multi-source Discovery: Feed, collections, search, category navigation

Real-Time Inventory: In-stock items surfaced dynamically

Personalized Browsing: Tailored to regional trends & local stock

Integrated Order Management

Step 4

Orders transmitted to Fynd's OMS for fulfillment. Klydo manages warehouse pick & pack. Webhooks push shipment and delivery events back to HMNow.

❯

Last-Mile Delivery & Real-Time Updates

Step 5

Last-mile logistics handled by Fynd. Klydo consumes tracking data to give full order visibility to the customer.

Order Picked Up: Status update triggered to HMNow via API

Live Tracking: Customer monitors rider on interactive map

Secure Delivery: OTP forwarded white-labeled as HMNow

Post-Order Returns

Step 6

Fully automated 6-step return loop — customer initiates return, eligibility auto-validated, reverse shipment created, pickup scheduled, QC at dark store, refund processed.

Average resolution: < 48 hours · No manual intervention · All white-labeled as HMNow

Security Guarantees for H&M
🔒

Data Residency — All H&M customer data (PII, payment, orders) lives in H&M's cloud account. Klydo has zero access to production data outside deployment windows.

📦

Code Isolation — Klydo Commerce Backend and Klydo AI run as isolated containers (Kubernetes) within H&M's infrastructure. No shared compute with Klydo's own platform or any other brand.

🌐

Network Security — All external communication over TLS 1.3. Internal services communicate within H&M's VPC. No public endpoints except H&M's own load balancer.

🛡

PII Boundary — H&M customer PII never leaves H&M's cloud. Only fulfillment data (SKU, quantity, dark store address) is sent to Fynd for dispatch.

💳

Payment Security — Klydo never touches card data. Payments flow through H&M's existing PCI-DSS compliant gateway. Klydo only receives payment confirmation tokens.

🔑

Access Control — Klydo engineers get time-boxed, audited access for deployments only. No standing access to H&M's infra. All access governed by H&M's IAM policies.

📋

Audit Trail — Every order, payment, refund, and inventory change is logged with immutable audit trails in H&M's database. Fully auditable by H&M's compliance team.

🔄

Software Updates — All updates delivered via Kubernetes rolling deployments within H&M's cloud. Zero downtime, no manual intervention. H&M retains full visibility and approval control over deployments.
